# 从 url 开始到面试结束

浏览器主要有五个进程：

- 浏览器进程：界面显示、用户交互、子进程管理，同时提供存储等功能
- Tab 渲染进程：包括排版引擎 blink 和 javaScript 引擎 V8 ，可以将 html、js、css 转换为页面
- GPU 进程：3D CSS、Chrome UI、网页
- 网络进程：网络资源加载
- 扩展或插件进程：运行扩展/插件，因为扩展/插件容易崩溃所以分离为进程

输入 url 之后，浏览器进程就开始处理导航流程

1. 首先浏览器进程根据用户输入的 url 解析，判断是否是搜索，如果不是搜索，则判断 url 的协议，以 http 为例，先尝试 https (扩展：<a href="../1http/90浏览器构建HTTP请求行时怎么确定HTTP版本的.html">浏览器构建 HTTP 请求行时怎么确定 HTTP 版本的</a>)

2. 然后 ipc 通信给网络进程，网络进程进行 dns 查询并缓存，然后对 ip 服务器建立 tcp 连接，接收到响应头后，解析响应头，处理缓存等，将数据发给浏览器进程(扩展：http，tcpip TODO)

3. 浏览器进程再通知渲染进程准备接收 ，如 html，并开始加载，并通知浏览器进程加载完成 (扩展：渲染,执行上下文,react fiber TODO)

4. 浏览器进程进行历史记录导航、缓存等处理

## 大致流程

- 浏览器进程接收用户输入，将 URL 转发给网络进程
- 网络进程开始 URL 请求
- 网络进程接收响应数据，将响应头数据转发给浏览器进程
- 浏览器进程接收响应头数据，将'准备渲染'消息发动到渲染进程
- 渲染进程接收'准备渲染'消息，与网络进程建立 IPC 数据管道接收 HTML 数据，网络进程接收完响应体数据后开始渲染
- 渲染进程发给浏览器进程'确认提交'表示已经渲染好
- 浏览器进程接收'确认提交'消息，开始更新页面

- 浏览器进程：处理用户输入，重定向，进程管理等
- 网络进程：发起 URL 请求，读取响应头和响应体数据
- 渲染进程：解析页面与资源

4. 如果服务器不支持 HTTPS 或者 TLS 连接失败，浏览器会回退到使用 HTTP 协议进行通信，并根据服务器返回的响应头中的"HTTP"字段来判断 HTTP 版本。
